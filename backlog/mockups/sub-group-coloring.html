Implementation notes
All styles use fill_rect_alpha() on the existing 32-bit ARGB DIB — no new Win32 APIs.
Styles can be combined (e.g., left bar + background tint). Config could offer a tab_color_style enum.
The dot (D) is the only one needing per-pixel circle writes — a 7x7 loop, trivial on the DIB.
"Trivial" = swap or add one fill_rect_alpha call. "Easy" = small per-pixel loop.
